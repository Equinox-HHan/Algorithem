# 暑假算法学习笔记

## 阶段1：

### day1-2:
1. 主定理：用于求解分治算法递推关系式时间复杂度：T(n)=aT(n/b)+f(n)
2. 三码问题：
    - 正数的原码反码补码都是一致的
    - 负数反码是原码除符号位全取反，补码是原码取反加一
    - 负数补码取反加一或者减一取反得到原码
    - 所有编码都是0正1负
3. 位运算：
    - `A^0=A,A^A=0`（xor相同取0不同取1,满足交换律和结合律）
    - `n&(n-1)`：去除二进制最右边的1
    - `n&(-n)`：获取二进制最右边的1
    - `~n&(n+1)`：获取最右边的0
    - 进行乘除运算和奇偶判断

### day3-4:
1. GCD(GCD(a,b)==GCD(b,a%b)),LCM(GCD(a,b)×LCM(a,b)=|a×b|)
2. 快速幂：时间复杂度logn
3. 埃氏筛：素数的倍数是合数（两个优化）
4. 同余原理（除法是特例）

### day5-6:
1. 基础二分查找
2. 寻找首末位特殊二分查找
3. 山峰位置二分查找`l<r`

### day9：

    1. 原地修改数组：移除元素，移动零，删除重复项
    2. 链表找中点或者找环
    3. 滑动窗口：连续子数组，连续子串

要么交换（需要保留元素）要么覆盖（可以删除元素）

基本上都是return slow

### day10-11：
1. 不定长滑动窗口：
    - 求解连续子数组/子串问题，求满足条件的最大/最小长度或者区间个数
    - right指针无脑右移添加元素到当前窗口，left指针负责右移使窗口处于合法状态
    - 最小：while符合条件，先收集答案然后left+=1
    - 最大：while不符合条件，left+=1最后收集答案
    - **核心思路**：
        1. 每次right+=1需要向窗口添加什么（更新某个状态）
        2. 什么时候需要进入while循环（违反题目条件）
        3. while里在left+=1之前需要丢弃什么（让窗口合法化）
        4. 是在while里返回ans还是在while外（都是在for里）
2. 定长滑动窗口：
    - 求长度为k的连续子数组最大和/均值，寻找特定长度的匹配子串/异位词
    - 核心逻辑：`right-left+1==k`
    - 标准模板：铺设初始窗口（特判）+整体平移滑动

### day12：
前缀和
1. 区间的静态检索和查询（一维+二维）p[i]表示第i个元素的前缀和，p[right+1]-p[left]表示left-right区间和
2. 前缀和+哈希表：求解满足某个条件的子数组p[j]-p[i]=k
3. 前缀和+差分：对区间进行修改并最后求和p[left]+=value,p[right]+=value
差分：和前缀和互为逆运算

## day13-14:
1. unordered_map,unordered_set（哈希表）map,set（红黑树严格弱序关系）
2. Hashmap：Key-Value用于哈希映射和计数，Hashset：Key用于去重和判断存在性 
3. 常见题型：
    - 寻找补数问题：匹配问题，两数之和，特定配对
    - 频次统计与字符状态过滤：异位词，频次topK，字符匹配
    - 哈希表+前缀和：和为K的子数组
    - 模式分组和Key的设计：相同特征的数据设计为统一的Key
    - 滑动窗口+哈希表
4. 基于哈希表的高级数据结构设计
- 技巧：
    1. 寻找众数用摩尔投票法：票数为零选新擂主，同类加票异类减票；不是严格众数最后要循环计数看是否大于一半
    2. 字母异位词：使用排序视角利用string自带sort()函数；`unordered_map<string,vector<string>>`是经典一对多
    3. 原地哈希：将输入数组当作unordered_map的极致空间优化
        - 主要考法：寻找重复值/缺失值，映射1-N的连续数
        - 将数组索引i看成哈希key，最终目标`nums[i]=i+1`
        - `if(nums[i]!=i+1)` `swap(nums[i],nums[nums[i]-1])`
        - 轻量级原地哈希：遍历数组对元素取|x|，`nums[|x|-1]=-nums[|x|-1]`，最后nums[i]>0则i+1不存在
